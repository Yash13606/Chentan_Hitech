# Chetan Hi-Tech — Procurement Platform

## What this project is
B2B industrial equipment procurement platform for Chetan Hi-Tech. Customers (facility managers, procurement teams) browse equipment, submit RFQs, receive branded PDF quotations, and manage their account via a portal. Sales team manages inquiries through a CRM. Admins control the full catalog and content.

**Live URL:** https://chentanhitech-production.up.railway.app  
**GitHub:** https://github.com/Yash13606/Chentan_Hitech  
**Stack:** Next.js 16.2.6 · React 19 · Prisma 7.8 · PostgreSQL (Railway) · Auth.js v5 · Cloudflare R2 · Resend · Tailwind v4

---

## Critical Next.js 16 rules (read before touching any file)
- `middleware.ts` is **dead** — auth middleware lives in `src/proxy.ts`
- Cache Components is OFF — use `unstable_cache` + `revalidateTag(tag, profile)` (2 args required)
- Turbopack is default — no webpack-only plugins
- `after()` is stable (not `unstable_after`) — import from `next/server`
- Every `generateStaticParams` that hits the DB must be wrapped in try/catch returning `[]`
- Every cached DB call on a static page needs `.catch(() => [])` to survive build-time placeholder credentials

## Architecture

```
src/
  proxy.ts              ← Next 16 middleware (NOT middleware.ts)
  app/
    page.tsx            ← RSC wrapper; fetches real data; renders HomeClient
    (auth)/             ← /login /signup /forgot
    (account)/          ← /portal/* — requires auth
    (admin)/            ← /admin/* — requires ADMIN role
    catalog/            ← product listing + filters
    products/[slug]/    ← product detail
    knowledge/          ← public article listing + [slug] detail
    cart/               ← cart page
    crm/                ← sales CRM kanban
    api/
      auth/[...nextauth]/
      ai/suggest/       ← Anthropic AI assistant (POST)
      health/           ← Railway healthcheck
      portal/bundles/   ← bundle data API
      admin/articles/[id]/ ← article fetch for edit
      uploads/presign/  ← R2 presigned PUT
  components/
    home/HomeClient.tsx ← "use client" — all motion + auth-aware nav
    cart/               ← add-to-cart-button, cart-merger
    providers.tsx       ← SessionProvider wrapper
    ui/                 ← industrial-* design system components
  server/               ← 'server-only' — NEVER imported by client code
    db.ts               ← Prisma singleton
    auth.ts             ← Auth.js v5 config + requireAuth() + requireRole()
    dal/                ← Data Access Layer (security boundary)
      products.ts
      articles.ts
    actions/            ← Server Actions (always call requireAuth first)
      _wrap.ts          ← requireAuth() + assertRole() helpers
      admin/            ← articles.ts, products.ts, csv-import.ts
      auth.ts, cart.ts, bundles.ts, rfq.ts, quotations.ts, crm.ts, consultations.ts, profile.ts
    services/
      ai.ts             ← Anthropic SDK (lazy import, graceful without key)
      audit.ts          ← writeAuditLog() — silent fail, never throws
      email/index.ts    ← Resend transactional emails
      pdf/              ← @react-pdf/renderer quotation PDF
      r2.ts             ← Cloudflare R2 S3 client
      rate-limit.ts     ← Upstash sliding window (graceful without keys)
      lead-scoring.ts   ← Pure function, no DB
    validators/auth.ts  ← Zod schemas
  lib/
    cart-store.ts       ← Zustand cart (localStorage for guests)
  generated/prisma/     ← Gitignored — regenerated by postinstall
```

## Data model (key entities)

| Model | Purpose |
|---|---|
| User | Auth + role. Roles: GUEST CUSTOMER SALES QUOTE_MGR ADMIN |
| Company | Linked to User. Has industry, leadScore |
| Product | Catalog item. Has slug, sku, price, specs (JSON), categoryId |
| Category | Self-referencing tree |
| Cart / CartItem | Guest (cookie) merged to DB on login |
| Inquiry | RFQ submission. Status: NEW→ASSIGNED→IN_REVIEW→QUOTED→WON/LOST |
| Quotation | Built by sales. Has PDF key on R2 |
| QuotationRevision | Version history |
| KnowledgeArticle | Blog/knowledge content. publishedAt=null means draft |
| Bundle / BundleItem | Saved equipment sets |
| Consultation | Booking |
| AuditLog | All admin/sales actions |
| Notification | In-app alerts |

## Auth & security model
- JWT sessions (no DB session table). Token contains `role`, `companyId`, `userId`
- 3-layer defence: proxy.ts coarse redirect → layout guard → DAL re-check
- `requireAuth()` in `src/server/actions/_wrap.ts` — first line of every Server Action
- DAL functions never return raw Prisma objects — only selected fields (passwordHash never exposed)
- Role check pattern: `const allowed: string[] = [Role.ADMIN, Role.SALES]; allowed.includes(session.user.role)`

## Key patterns to follow

**Cached DAL function:**
```ts
export const getPublishedArticles = unstable_cache(
  async (take = 10) => db.knowledgeArticle.findMany({ ... }),
  ["published-articles"],
  { tags: ["articles"], revalidate: 3600 }
);
// Always call with .catch(() => []) on static pages
```

**Server Action:**
```ts
export async function myAction(_prev: State, formData: FormData): Promise<State> {
  const session = await requireAuth();
  const allowed: string[] = [Role.ADMIN];
  if (!allowed.includes(session.user.role)) return { error: "Forbidden" };
  // ...
  await writeAuditLog(session.user.id, "ACTION", "Entity", entityId);
  revalidateTag("tag-name", "default");
  return { success: true };
}
```

**Prisma transactions** — callback form only (array form not supported in Prisma 7):
```ts
await db.$transaction(async (tx) => { ... });
```

## Design system
- Colors: Tailwind v4 `@theme` tokens — `foreground`, `background`, `primary`, `muted`, `border`, `card`
